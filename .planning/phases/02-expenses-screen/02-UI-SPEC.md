# Phase 2 — UI Design Contract

> Visual and interaction contract for the Expenses Screen (Android / Jetpack Compose + Material3).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn not applicable — Android project) |
| Preset | not applicable |
| Component library | Jetpack Compose + Material3 |
| Icon library | Material Symbols (`androidx.compose.material:material-icons-extended`) |
| Font | Roboto (Material3 default, system font on Android) |

**Notes:**
- Phase 1 used `Theme.AppCompat.DayNight.NoActionBar`. Phase 2 replaces the theme entirely with a Material3 `MaterialTheme` wrapper in Compose. The `themes.xml` entry remains as a bridge parent but the Compose entry point takes over via `setContent { MaterialTheme { ... } }`.
- Dark mode is deferred to v2 (UX-01). Phase 2 ships light theme only.
- Source: CONTEXT.md D-04, REQUIREMENTS.md, ROADMAP.md Phase 2 description.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | dp Value | Usage |
|-------|----------|-------|
| xs | 4dp | Icon-to-label gaps, chip inner horizontal padding supplement |
| sm | 8dp | Between chip rows; between summary card title and value |
| md | 16dp | Screen horizontal padding; item row vertical padding; card internal padding |
| lg | 24dp | Between the filter row and the expense list; below the summary card |
| xl | 32dp | Top-of-screen padding below status bar |
| 2xl | 48dp | Not used in Phase 2 |
| 3xl | 64dp | Not used in Phase 2 |

Exceptions:
- Touch target minimum: 48dp height for all tappable elements (FilterChip, date chips) — Android Material3 accessibility requirement. This overrides the row height calculation but does not change the spacing tokens.
- `LazyRow` category chips: `contentPadding = PaddingValues(horizontal = 16dp)`, `horizontalArrangement = Arrangement.spacedBy(8dp)` (sm token).
- Summary card corner radius: 12dp (Material3 `CardDefaults.shape` — not a spacing token, noted here for completeness).

## Typography

All sizes in `sp` (scale-independent pixels — Android standard, equivalent to `px` at 1x density).

| Role | Size | Weight | Line Height | Material3 Slot | Usage |
|------|------|--------|-------------|----------------|-------|
| Display | 28sp | SemiBold (600) | 1.2 | `headlineMedium` | Summary card total amount |
| Heading | 20sp | SemiBold (600) | 1.2 | `titleLarge` | Screen title "Wydatki"; summary card period label |
| Body | 16sp | Regular (400) | 1.5 | `bodyLarge` | Expense item description (primary row) |
| Label | 14sp | Regular (400) | 1.4 | `bodyMedium` | Expense item merchant + category (secondary row); date label; chip text |

**Constraints:**
- Exactly 2 weights used: Regular (400) and SemiBold (600). Do not introduce Medium (500) or Bold (700).
- Amount on expense list item: 16sp SemiBold (600) — same size as Body but elevated weight to draw the eye right.
- Do not use `sp` values outside 14, 16, 20, 28. Do not introduce 12sp, 18sp, or 24sp in Phase 2.

## Color

Material3 dynamic color is deferred to v2. Phase 2 uses a static seed-color palette.

**Seed color:** `#1B6CA8` (deep blue — financial/trust association, readable on white).

| Role | Hex / M3 Token | Usage |
|------|----------------|-------|
| Dominant (60%) | `#FFFFFF` / `M3.colorScheme.background` | Screen background |
| Secondary (30%) | `#F2F5F8` / `M3.colorScheme.surfaceVariant` | Expense list item row background (alternating is NOT used — all rows same); summary card surface; `LazyRow` chip row background strip |
| Accent (10%) | `#1B6CA8` / `M3.colorScheme.primary` | Selected `FilterChip` fill; active date quick-chip fill; summary card amount text |
| Destructive | `#B3261E` / `M3.colorScheme.error` | Error state message text and icon; not used on any action in Phase 2 (read-only screen) |

Accent reserved for:
1. Selected/active `FilterChip` background (category filter chips when tapped/active)
2. Active quick date chip fill ("Ten miesiąc" / "Poprzedni" when selected)
3. Total amount value text inside the summary card
4. Circular progress indicator during loading state

**Not accent:** unselected chips, list item text, card borders, icons in deselected state.

**Source:** CONTEXT.md (no color specified — Claude's discretion). Material3 baseline palette derived from seed `#1B6CA8`.

## Component Inventory

> Prescribed Compose components for this phase. Executor must use these exactly.

### ExpensesScreen (top-level Composable)
- `Scaffold` with `topBar = { TopAppBar(title = { Text("Wydatki") }) }`
- `Column` body: Summary Card → Date Quick Chips → Category Filter Row → Expense List

### Summary Card
- `Card(modifier = Modifier.fillMaxWidth().padding(horizontal = 16dp, vertical = 8dp))`
- Inside: `Column(padding = 16dp)` — period text (Label/14sp) on top, total amount (Display/28sp SemiBold) below
- Period text examples: `"Czerwiec 2026"` for current month, `"1 cze — 18 cze 2026"` for custom range
- Source: CONTEXT.md D-14, D-05

### Date Quick Chips (3 chips)
- `Row(horizontalArrangement = Arrangement.spacedBy(8dp), modifier = Modifier.padding(horizontal = 16dp))`
- 3x `FilterChip(selected = ..., onClick = ..., label = { Text(...) })`
- Labels (exact strings): `"Ten miesiąc"` | `"Poprzedni"` | `"Niestandardowy"`
- "Niestandardowy" triggers `DateRangePicker` in `ModalBottomSheet`
- Default selected on launch: `"Ten miesiąc"`
- Source: CONTEXT.md D-04, D-05, D-06

### Category Filter Row
- `LazyRow(contentPadding = PaddingValues(horizontal = 16dp), horizontalArrangement = Arrangement.spacedBy(8dp))`
- 13x `FilterChip(selected = category in selectedCategories, onClick = { toggle category })`
- Chip labels: Polish `displayName()` from `ExpenseCategory` enum (see table below)
- Initial state: no chip selected (all expenses visible)
- Multiple chips can be active simultaneously
- Source: CONTEXT.md D-07, D-08, D-09, D-10

**Category chip labels (exact Polish strings from backend enum):**

| Enum value | Chip label |

| HEALTH | Zdrowie |
| HOUSING | Mieszkanie |

| EDUCATION | Edukacja |

### Expense List
- `LazyColumn(modifier = Modifier.fillMaxSize())`
- Each item: `ExpenseListItem` Composable (see below)
- Items in reverse-chronological order (sort by `occurredDate` descending, then by `occurredAt` descending for same-day)
- Source: CONTEXT.md D-11, EXP-01

### ExpenseListItem Composable
- `Row(modifier = Modifier.fillMaxWidth().padding(horizontal = 16dp, vertical = 12dp))`
- Left column (weight = 1f):
  - Row 1: `Text(description, style = bodyLarge)` — 16sp Regular
  - Row 2: `Text("$merchant · $categoryDisplayName", style = bodyMedium)` — 14sp Regular, color = `onSurfaceVariant`
    - When `merchant` is null or blank: `Text(categoryDisplayName, style = bodyMedium)` — omit merchant entirely, no placeholder
- Right column (intrinsic width):
  - Row 1: `Text(formattedAmount, style = bodyLarge, fontWeight = SemiBold)` — 16sp SemiBold, right-aligned
  - Row 2: `Text(occurredDate, style = bodyMedium)` — 14sp Regular, right-aligned, color = `onSurfaceVariant`
- Amount format: `"${amount} PLN"` (e.g., `"123.45 PLN"`) — Polish formatting deferred to Phase 4 (APP-04)
- Date format: `"d MMM"` for current year (e.g., `"5 cze"`), `"d MMM yyyy"` for prior years (e.g., `"5 cze 2025"`)
- `HorizontalDivider` between items (1dp, color = `outlineVariant`)
- Source: CONTEXT.md D-11, D-12, D-13

### Loading State
- Full-screen `Box(contentAlignment = Alignment.Center) { CircularProgressIndicator(color = primary) }`
- Shown while `ExpensesUiState.Loading`

### Error State
- Full-screen `Column(horizontalAlignment = Alignment.CenterHorizontally, verticalArrangement = Arrangement.Center)`
- `Icon(Icons.Default.ErrorOutline, tint = error)` — 48dp
- `Text("Nie udało się wczytać wydatków", style = titleLarge, textAlign = Center)`
- `Text("Sprawdź połączenie z internetem i spróbuj ponownie.", style = bodyMedium, textAlign = Center)`
- `Button(onClick = { viewModel.retry() }) { Text("Spróbuj ponownie") }`
- Source: EXP-01, REQUIREMENTS.md APP-03 (pattern established here for Phase 4 to reuse)

### Empty State
- Full-screen `Column(horizontalAlignment = Alignment.CenterHorizontally, verticalArrangement = Arrangement.Center)`
- `Icon(Icons.Default.ReceiptLong, tint = primary)` — 64dp
- `Text("Brak wydatków", style = titleLarge, textAlign = Center)`
- `Text("Nie znaleziono wydatków dla wybranego okresu i filtrów.", style = bodyMedium, textAlign = Center)`

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Launch | Screen opens | Load current month expenses; show Loading state, then Success or Error |
| Tap "Ten miesiąc" chip | User taps | Select chip; deselect others; reload expenses for first-of-month → today |
| Tap "Poprzedni" chip | User taps | Select chip; deselect others; reload expenses for previous calendar month |
| Tap "Niestandardowy" chip | User taps | Open `DateRangePicker` in `ModalBottomSheet`; chip stays unselected until confirmed |
| Confirm date range | Tap "Zastosuj" in picker | Close picker; select "Niestandardowy" chip; reload expenses for chosen range; update summary card label |
| Cancel date range | Tap "Anuluj" or swipe down | Close picker; restore previous selection; no data reload |
| Tap category chip (unselected) | User taps | Add category to active filter set; refilter displayed list client-side; update summary card total |
| Tap category chip (selected) | User taps | Remove category from active filter set; refilter displayed list client-side; update summary card total |
| All category chips deselected | (state) | Show all loaded expenses; summary total = sum of all expenses in date range |
| Tap "Spróbuj ponownie" | User taps error CTA | Re-trigger API call with same date range; show Loading state |
| Scroll expense list | User scrolls | Standard `LazyColumn` fling scroll; date chips and category row remain fixed (not sticky) |

**State management:** All filter state lives in `ExpensesViewModel`. Composable is stateless — it reads `StateFlow<ExpensesUiState>` and `StateFlow<FilterState>` only.
